# 무료 보안 설정 적용

웹사이트 코드는 GitHub Pages 배포 후 자동 반영되지만 Firestore 보안 규칙은 Firebase에서 한 번 게시해야 합니다.

1. Firebase Console에서 `dankook-re` 프로젝트를 엽니다.
2. **Firestore Database → 규칙**으로 이동합니다.
3. 저장소의 `firestore.rules` 내용을 전부 복사해 기존 규칙과 교체합니다.
4. **게시**를 누릅니다.

적용 후 동작:

- 관리자 또는 `users/{uid}.status == "approved"`인 계정만 사이트 데이터를 이용합니다.
- 승인되지 않은 계정은 로그인에 성공하더라도 보호 페이지가 열리지 않습니다.
- 승인된 계정이 새로운 브라우저·기기에서 로그인하면 `securityEvents`에 기록됩니다.
- 보안 기록은 관리자만 읽고 확인·삭제할 수 있습니다.
- 비로그인 요청과 규칙에 명시되지 않은 모든 데이터 접근은 거부됩니다.

주의: GitHub Pages의 HTML·CSS·기본 이미지는 정적 파일이므로 공개됩니다. 개인정보는 Firestore 또는 접근이 제한된 저장소에만 두어야 합니다.
