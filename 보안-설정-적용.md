# 무료 보안 설정 적용

웹사이트 코드는 GitHub Pages 배포 후 자동 반영되지만, Firestore 보안 규칙은 Firebase에서 별도로 게시해야 합니다.

1. Firebase Console에서 `dankook-re` 프로젝트를 엽니다.
2. **Firestore Database → 규칙**으로 이동합니다.
3. 저장소의 `firestore.rules` 전체 내용을 복사해 기존 규칙과 교체합니다.
4. **게시**를 누릅니다.

적용 후 동작:

- 관리자 또는 `users/{uid}.status == "approved"`인 계정만 사이트 데이터를 이용합니다.
- 승인되지 않은 계정은 로그인에 성공하더라도 보호 페이지를 열 수 없습니다.
- 새 기기 로그인 기록과 관련 알림은 생성하지 않습니다.
- 설정에서 비밀번호를 변경하면 `passwordChangeEvents`에 계정 이메일과 변경 시각만 기록됩니다.
- 비밀번호 내용은 저장하지 않으며, 변경 내역은 관리자만 읽거나 삭제할 수 있습니다.
- 이 기능을 적용한 이후 웹사이트 설정에서 변경한 내역만 기록됩니다.
- 비로그인 요청과 규칙에 명시되지 않은 모든 데이터 접근은 거부됩니다.

주의: GitHub Pages의 HTML·CSS·기본 이미지 등 정적 파일은 공개됩니다. 개인정보는 Firestore처럼 접근이 제한된 저장소에서만 다뤄야 합니다.
