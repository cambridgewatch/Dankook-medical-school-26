# 실제 코드 기준 데이터 구조표

아래 내용은 첨부 ZIP의 JavaScript와 `firestore.rules`를 직접 대조해 작성했습니다. `firebase-init.js`와 첨부파일 모듈은 ZIP에서 제외되어 있으므로, 첨부파일 내부 객체의 세부 키는 확인할 수 없습니다. Worker에서는 첨부파일 배열의 **개수만** 사용합니다.

## 1. Worker가 읽는 컬렉션

| Firestore 경로 | 실제 코드에서 확인된 필드 | 형식·의미 | Worker 반환 |
|---|---|---|---|
| `calendarEvents/{eventId}` | `date`, `endDate`, `text`, `detail`, `type`, `attachments`, `by`, `createdAt`, `updatedAt` | 전체 일정 | 허용 |
|  | `kind`, `sourceId`, `hidden`, `version`, `migratedAt` | 기본 일정 이관·override 제어 필드 | 문서 판별에만 사용, 외부 미반환 |
| `calendarPersonal/{ADMIN_UID}/events/{eventId}` | `date`, `endDate`, `text`, `detail`, `type`, `attachments`, `by`, `createdAt`, `updatedAt` | 관리자 개인 일정 | 허용. 경로 UID와 `by`는 미반환 |
| `notices/{noticeId}` | `title`, `detail`, `tag`, `attachments`, `createdAt`, `updatedAt`, `pinned`, `pinnedAt` | 공지 | 허용. 첨부는 개수만 |
| `alerts/{alertId}` | `type`, `title`, `detail`, `date`, `text`, `noticeId`, `calendarEventKey`, `createdAt`, `updatedAt` | 공지·일정 알림 | 제목·내용·날짜·유형·시각만 허용 |
| `polls/{pollId}` | `title`, `description`, `options[]`, `allowVoteChange`, `showResultsBeforeClose`, `allowOptionAdd`, `creatorUid`, `creatorName`, `closed`, `createdAt`, `updatedAt`, `closedAt` | 투표 본문·설정 | 작성자 UID·이름 제외 후 허용 |
| `polls/{pollId}/votes/{uid}` | `optionId`, `updatedAt` | 실제 투표 | **문서 수만 참여 인원으로 계산** |

## 2. 일정 분류 값

실제 `calendar.js`에서 확인된 값:

| 값 | 의미 |
|---|---|
| `acad` | 학사 |
| `exam` | 시험 |
| `event` | 행사 |
| `vac` | 방학 |
| `holi` | 공휴일 |
| `todo` | 할 일 |
| `assignment` | 과제 |
| `meet` | 모임 |
| `etc` | 기타 |

## 3. 공지 분류 값

실제 `notices.js`에서 확인된 값:

| 값 | 의미 |
|---|---|
| `notice` | 필독 |
| `event` | 행사 |
| `acad` | 학사 |
| `info` | 안내 |

## 4. 투표 옵션 구조

```json
{
  "options": [
    {
      "id": "내부 옵션 ID",
      "text": "화면에 표시되는 선택지"
    }
  ]
}
```

Worker는 `id`를 외부에 반환하지 않고 `text`를 `label`로 복사합니다. 개별 선택 결과나 선택지별 득표수는 반환하지 않습니다.

## 5. 내부에서만 조회하는 컬렉션

- `members`: 공지·일정·투표의 자유 입력 문장에 포함된 동기 실명을 `[이름]`으로 치환하기 위해 `name` 값만 사용합니다. 명단·문서 ID·이름은 응답에 포함하지 않습니다.

## 6. 조회하지 않는 컬렉션

- `users`
- `passwordChangeEvents`
- `polls/{pollId}/voters`
- `photos`
- `timetableGlobal`
- 다른 사용자의 `calendarPersonal`
- 다른 사용자의 `timetablePersonal`

## 7. 확인 한계

- `attachments.js`가 ZIP에 없으므로 첨부 객체 내부 필드는 검증하지 못했습니다.
- `firebase-init.js`가 ZIP에 없으므로 실제 관리자 값과 외부 서비스 키는 검증하지 않았습니다.
- Firestore 실제 저장 데이터 샘플은 포함되지 않아, 선택 필드의 누락·과거 스키마 변형 가능성은 Worker의 기본값 처리로 방어했습니다.
- 명단에 없는 이름이나 별칭은 자동으로 식별할 수 없으므로 `REDACT_TERMS` Secret에 추가해야 합니다.
