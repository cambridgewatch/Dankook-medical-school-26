# 개인정보·비용·보안 점검

## 배포 전 필수

- 브리핑 전용 Firebase 서비스 계정을 만들고 Firestore 읽기 권한만 부여합니다.
- `FIREBASE_SERVICE_ACCOUNT`, `ACTION_AUTH_TOKEN`, `ADMIN_UID`, `REDACT_TERMS`는 Cloudflare Secret에만 저장합니다.
- 실제 Secret을 GitHub, Worker 코드, GPT 지침, 채팅에 붙여 넣지 않습니다.
- 맞춤형 GPT는 나만 사용하도록 비공개로 유지합니다.
- Action 인증은 Bearer API Key를 사용하고 충분히 긴 무작위 토큰을 등록합니다.
- Worker Route가 기존 비밀번호 재설정 Route와 겹치지 않는지 확인합니다.

## 응답 개인정보

- `members`는 알려진 실명을 치환하는 내부 목적으로만 읽고 외부에 반환하지 않습니다.
- 이메일 패턴, UID 필드, 작성자 필드, 투표자 정보는 응답에서 제외합니다.
- 명단에 없는 별칭이나 이름은 자동으로 판단할 수 없으므로 `REDACT_TERMS`에 추가합니다.
- 첨부파일 URL과 실제 파일 내용은 반환하지 않고 개수만 제공합니다.
- 자유 입력 내용에 새로운 개인정보 형식이 생기면 치환 규칙을 갱신합니다.

## 비용과 사용량

- Worker는 요청당 Firestore 컬렉션 여러 개와 진행 중 투표의 `votes`를 읽습니다.
- 60초 응답 캐시를 사용해 반복 호출을 줄입니다.
- 맞춤형 GPT 질문 한 번에 Action을 반복 호출하지 않도록 GPT 지침을 유지합니다.
- Cloudflare와 Firebase 사용량 화면을 배포 초기에는 주기적으로 확인합니다.

## 사고 대응

- Action 토큰이 노출되면 즉시 새 토큰으로 교체합니다.
- 서비스 계정 JSON이 노출되면 해당 키를 폐기하고 새 키를 발급합니다.
- 예상하지 못한 개인정보가 응답에 나타나면 Route를 비활성화한 뒤 치환 규칙을 수정합니다.
- Worker 로그에 Authorization 헤더나 서비스 계정 내용을 출력하지 않습니다.
