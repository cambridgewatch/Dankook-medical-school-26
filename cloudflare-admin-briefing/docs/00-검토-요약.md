## 설계 결정

1. 기존 비밀번호 재설정 Worker는 건드리지 않았습니다.
2. 새 Worker는 서비스 계정 OAuth로 Firestore REST API를 읽습니다.
3. Action 인증은 별도 Bearer Secret을 사용합니다.
4. 응답은 allowlist 방식으로 새 객체를 구성합니다.
5. `ADMIN_UID` 경로의 개인 일정만 읽습니다.
6. 투표는 `votes` 문서 개수만 계산합니다.
7. 사진과 첨부 URL은 반환하지 않습니다.
8. 한국 시간 기준으로 오늘 포함 최대 7일만 반환합니다.
9. 60초 캐시와 요청 제한을 적용합니다.
10. `members`의 이름은 자유 입력 문장에서 실명을 가리는 내부 치환 목록으로만 사용하고 응답에는 넣지 않습니다.
