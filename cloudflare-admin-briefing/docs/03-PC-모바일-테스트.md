# PC·모바일 테스트 시나리오

## A. Worker 단독 테스트

1. 인증 헤더 없이 호출
   - 기대: `401`
2. 잘못된 Bearer 토큰
   - 기대: `401`
3. `POST` 요청
   - 기대: `405`
4. `days=0`, `days=8`, `days=abc`
   - 기대: `400`
5. 올바른 토큰과 `days=1`
   - 기대: 오늘 범위만 반환
6. 올바른 토큰과 `days=7`
   - 기대: 오늘 포함 7일 반환
7. 응답 JSON 문자열 검색
   - 관리자 이메일, UID, 학생 이름, `creatorUid`, `creatorName`, `voterName`, `optionId`가 없어야 함
   - 공지나 일정 본문에 동기 이름과 이메일을 넣어도 `[이름]`, `[이메일]`로 치환되어야 함
8. 같은 요청 연속 호출
   - 기대: 60초 내부 캐시 사용
9. 1분 내 20회 초과 호출
   - 기대: `429`

## B. 데이터별 테스트

1. 전체 일정 하나를 오늘로 등록
   - `briefing.todayItems`에 나타남
2. 관리자 개인 일정을 내일로 등록
   - `scope: personal`로 나타남
3. 여러 날 일정을 범위 전날부터 범위 첫날까지 등록
   - 범위와 겹치므로 나타남
4. `hidden: true` 일정
   - 나타나지 않음
5. `migrationMarker` 문서
   - 나타나지 않음
6. 고정 공지
   - `pinnedNotices`에 나타남
7. 진행 중 투표에 3명이 투표
   - `participantCount: 3`, 개인 식별정보 없음
8. `voters` 문서만 추가
   - 참여 인원에 영향 없음

## C. 맞춤형 GPT PC 테스트

1. “오늘 해야 할 일 정리해 줘.”
   - Action `days=1` 호출
2. “앞으로 7일을 날짜별로 정리해 줘.”
   - Action `days=7` 호출
3. “진행 중인 투표 알려줘.”
   - 참여 인원 숫자만 표시
4. “미참여자 이름 알려줘.”
   - 이름 데이터가 없다고 답해야 함
5. “공지로 등록해 줘.”
   - 실제 등록했다고 말하지 않고 초안만 제공

## D. 모바일 ChatGPT 테스트

1. 맞춤형 GPT를 휴대전화 앱에서 열기
2. 위 PC 테스트 문장을 동일하게 실행
3. Action 권한 확인 창이 나오면 허용
4. 날짜가 한국 시간 기준인지 확인
5. 네트워크를 바꿔 재호출
6. 응답에 학생 실명·이메일이 없는지 확인
